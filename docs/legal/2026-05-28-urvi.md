# Legal Memo — `urvi` branch (auth proxy/middleware change)

**Date:** 2026-05-28
**Filed by:** PM-Agent (advisory, non-blocking)
**Branch:** `urvi` (commit 92f858b — "unauthorised user test email fixed")
**Reviewer routing:** Adv. Aman Kaushik (no immediate action — informational)

## Summary
Branch changes `proxy.ts` (request routing / auth gate) and the
`/api/assessment-booking` route (which sends a booking confirmation email
containing patient-supplied PII: name, contact, booking ref).

## Surfaces touched
- `proxy.ts` — adds an early-return for public-auth paths (`isPublicAuth(pathname) → false`)
  and excludes `/api/*` from the middleware matcher entirely. This narrows the auth gate;
  the public-auth list is unchanged.
- `app/api/assessment-booking/route.ts` — converts a fire-and-forget `sendMail(...).catch` into
  a structured `logger.error` on failure with `emailNormalised` and `requestId`. Email content
  unchanged.
- `scratch/test_email_directly.ts` — diagnostic-only change (test recipient swap).

## Risk read
- **Auth scope narrowing on `/api/*`:** the matcher change pulls all API routes
  out of the middleware. Any auth currently enforced *inside* the handlers must
  remain. None of the route handlers under `app/api/auth/**` or `app/api/patient/**`
  are modified by this branch, so no regression to per-route auth is introduced.
- **PHI logging:** `emailNormalised` is the patient's email and qualifies as
  personal data under DPDP. Logging at `error` level is acceptable provided
  log retention follows the org's PII retention policy. No clinical/PHI fields
  (lab results, diagnosis, etc.) are added to the log payload.
- **No new consent flow, billing, or LabResult/Invoice/Consent model touch.**

## Recommendation
Merge as part of Sprint 1 close. No counsel sign-off required. Confirm in Sprint 2
that the `lib/logger.ts` PII redaction list includes `emailNormalised` /
generic email fields.
